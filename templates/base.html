Bedrock's layer of CSS love

Welcome to a brief overview of the new style for Mozilla websites — we call it Sandstone. This guide is meant to highlight some of the key elements and help promote a cohesive look for any site that wants to reflect the standard Mozilla style. Feel free to get in touch with the Creative team in #fistbump on IRC if anything is unclear, if you need some pointers on adapting this style or if you'd like us to have a go at designing a fresh adaptation for you. We're here as creative consultants, so by all means let us help if we can! Enjoy!

Less framework

Responsive grid

Responsive Web design is now a key ingredient for smart layouts that play nice with the wide range of screen ratios and resolutions available on today's Internet-enabled devices. CSS media queries empower designers and developers to customize layouts that best address the on-screen real estate for all of the smartphone/tablet/desktop platforms. Less Framework (combined with Less CSS) will be our scaffolding grid of choice going forward.

Grid notes

  • CSS media queries–based layout for smartphone, landscape smartphone, tablet and desktop
  • Maximum width on smartphone is 252px (3 grid columns)
  • Ideal main body copy width on tablet and desktop is 528px (6 columns)
  • .org style uses vertical spacing gutters of multiples of 24px to separate sections

    With 200+ sites, 150 blogs and many other various Web properties, the Mozilla universe is massive and can be difficult to navigate. Tabzilla is there to help you find what you're looking for. A lot of you have probably become pretty familiar with the grey tab in the upper right corner of Mozilla sites, which has been helpful in identifying Mozilla properties from a branding perspective. Now we’re excited to roll out the next phase of the universal tab — along with a new look and new functionality — as part of the One Mozilla project.

    Typefaces

    We've set a new standard for our headline font on Mozilla websites — Open Sans. Other fonts can be used for specific projects/campaigns/landing pages/apps, since we don't want to enforce one specific font for all sites, but this is the default Mozilla style. We're pretty excited about switching over to a fully open font as our standard and we hope the community is as well.

    back to top

    Type CSS

    @font-face { font-family: "OpenSans"; src: url(OpenSans-Light.ttf) format("truetype"); }
    body {font: 16px/24px Georgia, serif;}

    .huge, .large, h1, h2, h3, h4 {font-family: 'OpenSans', "Lucida Sans", "Lucida Grande", "Lucida Sans Unicode", Verdana, sans-serif; font-weight: normal; text-shadow: 0px 1px 0px rgba(255,255,255,0.75);}

    .huge {font-size: 108px; letter-spacing: -4px; line-height: 100%;}
    .large {font-size: 72px; letter-spacing: -3px; line-height: 100%;}
    h1 {font-size: 48px; letter-spacing: -2px; line-height: 100%;}
    h2 {font-size: 32px; letter-spacing: -1px; line-height: 100%;}
    h3 {font-size: 28px; letter-spacing: -0.5px; line-height: 100%;}
    h4 {font-size: 24px; letter-spacing: -0.25px; line-height: 100%;}
    .small, small { font-size: 12px; line-height: 100%;}

    Backgrounds

    The vertical stripes live on! There has been some discussion around the use of the vertical stripes as the default background for our sites, so we've evolved the look to much more subtle lines and a fine grain. With the PSD supplied, a custom color gradient can be made for your sites if you'd like to stay in line with this default style. Again, other backgrounds can be used for specific projects/campaigns/landing pages/apps.
